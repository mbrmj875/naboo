# ديون تقنية — مسودة داخلية

## مزامنة السحابة (`CloudSyncService`)

**سباق الرفع بين جهازين:** قد يُستبدل صف `app_snapshots` لأحدث رفع ناجح بين جهازين يكتبان تقريباً في نفس اللحظة؛ لا يوجد قفل تفاؤلي (optimistic locking) على السيرفر بعد. تحسينات «سحب قبل رفع» و«استعلام خفيف ثم payload» تقلّل النافذة والعبء الشبكي لكن لا تلغي هذا السباق.

**اتجاه لاحق محتمل:** رفض أو إعادة محاولة الرفع إذا لم يعد `updated_at` على السحابة يطابق ما قرأه العميل (`WHERE updated_at = last_seen`)، أو خدمة رفع تدمج على السيرفر.
