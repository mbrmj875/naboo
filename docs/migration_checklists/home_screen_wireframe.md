# Wireframe: الشاشة الرئيسية للأجهزة الستة

> **الملف المُرحَّل:** `lib/screens/home_screen.dart`
> **المرجع:** [`home_screen_checklist.md`](./home_screen_checklist.md) + [`screen_migration_playbook.md`](../screen_migration_playbook.md)
> **الاتجاه:** كل الـ wireframes هنا بـ RTL (العربية). اليمين هو “البداية”.

---

## رموز الـ Wireframe

```
┌─┐  حدود حاوية
│ │  محتوى عمودي
│=│  AppBar (لون primary)
│-│  Banner (شريط معلومات)
│ │  Body
│o│  Bottom Navigation Bar
[icon]  أيقونة
[txt]   نص
[btn]   زر
(...)   محتوى مخفي/قابل للسحب
═══     فاصل أفقي
║       فاصل عمودي
```

## 1. `phoneXS` (العرض < 360 أو الارتفاع < 600)

> **المثال:** iPhone SE 1st gen (320×568)، نوافذ مقسومة على Android.
> **التركيز:** الحد الأقصى من المحتوى، صفر زخارف.

├──────────────────────────────────────┤

│ │  مرحبًا، علي 👋                  │ │
│ │  ملخص اليوم                      │ │
│ │  ─────────                       │ │
│ │  ┌──────────────────────────┐    │ │
│ │  │ 💰  الصندوق              │    │ │
│ │  │     559,500 د.ع          │    │ │
│ │  └──────────────────────────┘    │ │
│ │  ┌──────────────────────────┐    │ │
│ │  │ 📦  المخزون              │    │ │
│ │  │     3554 صنفًا           │    │ │
│ │  └──────────────────────────┘    │ │
│ │  ┌──────────────────────────┐    │ │
│ │  │ 🧾  الفواتير             │    │ │
│ │  │     معلّقات: 0           │    │ │

│ │  │ صورة │  │ صورة │              │ │

### القرارات للـ phoneXS

| العنصر | القرار |
|---|---|
| AppBar height | 56dp (افتراضي Material) |
| Search bar | **أيقونة فقط** في الـ AppBar (تفتح Overlay كامل عند الضغط) |
| Permission banner | **أيقونة Tooltip** فقط (لا Banner، توفير للمساحة) |
| BottomNav slots | **4** (الصندوق، الفواتير، المخزون، المزيد) |
| العملاء | في قائمة “المزيد” |
| QuickActions | **محذوف نهائيًا** |
| Dashboard columns | **1 عمود** |
| Pinned products | تمرير أفقي (carousel) |
| Floating Calculator | **في قائمة “المزيد” فقط** (ليس FAB) |
| Breadcrumb | **مخفي** |
| Mac-style floating panels | معطلة (تفتح كـ push عادي) |
| Glassmorphism | معطلة (أداء) |

## 11. ملاحظات تنفيذية للـ AdaptiveScaffold

> هذه الـ wireframes تكشف 3 مشاكل صغيرة في `AdaptiveScaffold` الحالي يجب معالجتها:

1. **`AdaptiveScaffold` يدعم slot `appBar` فقط** — لكن نحتاج slot إضافي لـ `permissionBanner` (يظهر بين الـ AppBar والـ body).
   - **الحل المقترح**: إضافة `Widget? bottomBanner` slot أو استخدام `bottom:` من `PreferredSize` في الـ AppBar.

2. **`AdaptiveScaffold` لا يدعم Side Info Panels** (desktopSM, desktopLG).
   - **الحل المقترح**: إضافة `Widget? endDrawer` و `Widget? endDrawerLG` للـ desktop variants.

3. **`AdaptiveScaffold` لا يدعم Hover للديسكتوب** كنمط جاهز.
   - **الحل المقترح**: يبقى مسؤولية المحتوى نفسه (الـ Dashboard cards). الـ scaffold لا يحتاج معرفة بذلك.

> **قرار**: قبل بدء كتابة الكود، **يجب إضافة `bottomBanner` و`endPanels` للـ AdaptiveScaffold**. PR صغير قبل ترحيل الشاشة الرئيسية.
