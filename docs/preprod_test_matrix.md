# Checklist اختبارات ما قبل الإنتاج (Pre-Prod)

هذا الـ checklist **إلزامي** قبل تفعيل أي Feature Flag أو ترحيل عام للمشتركين.

> مهم: هذا الملف ليس “توثيقاً” فقط — هو ورقة تنفيذ. ضع ✅/❌ لكل سيناريو وسجّل الملاحظات.

## القاعدة الذهبية

- لا تُغيّر البيانات على السحابة أثناء الاختبار إلا عبر التطبيق/لوحة الإدارة.
- في كل سيناريو سجّل:
  - الجهاز/المنصة
  - الوقت (UTC)
  - النتيجة داخل التطبيق
  - ما ظهر في `security_audit_logs` (عبر لوحة الإدارة فقط)
  - حالة `account_devices` و`licenses` (عبر لوحة الإدارة فقط)

## بيئة الاختبار

- حساب Google واحد للاختبار.
- 3 أجهزة (اكتب وصفها عند التنفيذ):
  - **A**: ________________________
  - **B**: ________________________
  - **C**: ________________________
- تأكد من تطبيق SQL في Supabase:
  - `admin-web/supabase/device_limit_functions.sql`
  - `admin-web/supabase/security_audit_logs.sql`
  - `admin-web/supabase/app_server_time.sql`
